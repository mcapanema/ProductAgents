# GitHub Actions CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions workflow that lints, format-checks, and tests the project on every push to `main` and every pull request, gated on a 90% coverage floor.

**Architecture:** A single `ci.yml` workflow runs one `test` job on `ubuntu-latest`. It installs `uv` via `astral-sh/setup-uv` (with caching), syncs the locked dependencies, then runs `ruff check`, `ruff format --check`, and `pytest`. Ruff is added as a dev dependency and the codebase is brought into compliance first so the workflow is green on its first run. The coverage gate is enforced inside pytest via `--cov-fail-under`, so it works identically locally and in CI.

**Tech Stack:** GitHub Actions, uv, ruff, pytest + pytest-cov, Python 3.14.

## Global Constraints

- Python pinned to `3.14` via `.python-version` (do not hardcode a different version in CI; let `setup-uv` read the pin).
- Package/dependency management is **uv** only — never invoke `pip`, `conda`, or `poetry`.
- The full test suite runs offline with a fake model; CI requires **no API keys or secrets**.
- `uv.lock` is committed and authoritative — CI installs with `--frozen`; any dependency change must update and commit the lock.
- Current coverage is 94% (`uv run pytest` TOTAL). The coverage gate is set to **90%** (a deliberate buffer below current).
- This work belongs on a feature branch off `main` (e.g. `ci/github-actions`), not on the existing `docs/claude-md-and-readme` branch.

### Task 2: Enforce a 90% coverage gate

Wires the coverage floor into pytest's default options so the same gate applies locally and in CI.

**Files:**
- Modify: `pyproject.toml` (`[tool.pytest.ini_options].addopts`)

**Interfaces:**
- Consumes: the passing suite from Task 1.
- Produces: `uv run pytest` fails when total coverage drops below 90%. Task 3's workflow relies on this — its test step runs plain `pytest` and inherits the gate.

- [ ] **Step 1: Add the coverage-floor flag to pytest addopts**

In `pyproject.toml`, change the `addopts` line under `[tool.pytest.ini_options]` from:
```toml
addopts = "--cov=productagents --cov-report=term-missing --cov-report=html"
```
to:
```toml
addopts = "--cov=productagents --cov-report=term-missing --cov-report=html --cov-fail-under=90"
```

- [ ] **Step 2: Verify the suite still passes with the gate active**

Run:
```bash
uv run pytest
```
Expected: PASS — "35 passed", and coverage (94%) is above the 90% floor so pytest exits 0 (no "Coverage failure" line).

- [ ] **Step 3: Verify the gate actually fails below threshold**

Run a one-off with an absurd floor to prove the gate is wired (does not modify any file):
```bash

```
Expected: FAIL — pytest exits non-zero with a line like "Coverage failure: total of 94 is less than fail-under=100".


### Task 3: Add the GitHub Actions CI workflow

Creates the workflow that runs the now-passing lint, format, and test commands on pushes to `main` and on pull requests.

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: the lint-clean, formatted, gated repo from Tasks 1–2. The workflow runs `uv run ruff check .`, `uv run ruff format --check .`, and `uv run --frozen pytest` — all of which now pass locally.
- Produces: a CI check that runs on every PR (this branch's own PR will be its first live run).

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/ci.yml` with exactly this content:
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Check out the repository
        uses: actions/checkout@v5

      - name: Install uv
        uses: astral-sh/setup-uv@v8
        with:
          enable-cache: true

      - name: Install dependencies
        run: uv sync --frozen

      - name: Lint
        run: uv run ruff check .

      - name: Format check
        run: uv run ruff format --check .

      - name: Test
        run: uv run --frozen pytest
```
(`setup-uv` reads `.python-version` and installs Python 3.14 automatically — no `actions/setup-python` and no hardcoded version needed.)

- [ ] **Step 2: Validate the YAML parses**

Run:
```bash
uv run python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/ci.yml')); print('ci.yml: valid YAML')"
```
Expected: prints `ci.yml: valid YAML` and exits 0.
(If PyYAML is not importable in the project venv, validate with `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"` instead, or skip to Step 3 — the live PR run is the authoritative check.)

- [ ] **Step 3: Dry-run the exact CI commands locally**

Run each command the workflow runs and confirm all pass:
```bash
uv sync --frozen && uv run ruff check . && uv run ruff format --check . && uv run --frozen pytest
```
Expected: dependency sync succeeds, ruff reports no issues, format check passes, and pytest reports "35 passed" above the 90% floor.

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: run lint, format check, and tests on GitHub Actions"
```

- [ ] **Step 5: Push and open the PR to see CI run live**

```bash
git push -u origin HEAD
gh pr create --fill
```
Expected: a PR is created and the `CI / test` check starts running. Confirm it goes green with:
```bash
gh pr checks --watch
```
Expected: `CI / test` reports `pass`.

## Notes

- **Why no separate `actions/setup-python`:** `astral-sh/setup-uv` installs the Python version from `.python-version`, keeping a single source of truth for the version (the Global Constraint).
- **Why the gate lives in pytest, not the workflow:** putting `--cov-fail-under=90` in `addopts` means `uv run pytest` enforces the same floor on a developer's machine as in CI — no drift between local and CI behavior.
- **Hardening option (not required):** for stricter supply-chain hygiene you can pin `astral-sh/setup-uv@v8` and `actions/checkout@v5` to full commit SHAs and let Dependabot manage updates.
